# QuietScribe v0.1.2

This release fixes the UI becoming unresponsive during transcription and adds live transcript streaming.

## Fixed

- Replaced the blocking worker stdout read loop with a fully async `ReadLineAsync` loop.
- The app should no longer show "Not responding" while the worker is loading models or transcribing.

## Added

- The Python worker now emits each transcript segment as it is produced.
- The transcript editor updates live during transcription instead of waiting for the final result.

## Notes

- There can still be a short quiet period while the selected model loads and the first segment is decoded, but the Windows UI should remain responsive during that time.
